Master configurations for generating Training and Assessment Strategy documents

What is a TAS Template?

A TAS Template is a master configuration that defines how Training and Assessment Strategy documents should be structured, formatted, and generated. Templates are reusable blueprints that can generate multiple TAS documents for different qualifications.

üéØ Key Purpose

Templates separate structure and rules (template configuration) from content (generated TAS documents). This means you can:

  • Create a template once, use it for hundreds of qualifications
  • Update the template and have changes reflect in new TAS documents
  • Maintain consistency across your RTO's documentation
  • Comply with ASQA requirements systematically

Template Fields

Field Type Description
template_name String Display name (e.g., "Business Services Certificate III")
template_code String Unique identifier (e.g., "BUS_CERT_III_2024")
template_type Choice general, trade, business, health, creative, hospitality, technology, education
aqf_level Choice certificate_i, certificate_ii, certificate_iii, certificate_iv, diploma, advanced_diploma, vocational_graduate_certificate, vocational_graduate_diploma, graduate_diploma
description Text What this template is for and when to use it
structure JSONField Complete structural configuration (formatting, validation, settings)
gpt_prompts JSONField Template-level AI generation prompts (optional overrides)
is_active Boolean Whether this template is available for use

AQF Levels

Templates must specify the Australian Qualifications Framework (AQF) level:

AQF Level Code Typical Duration Characteristics
Certificate I certificate_i 12-24 weeks Foundation skills, high support, basic tasks
Certificate II certificate_ii 24-52 weeks Basic operational knowledge, routine tasks
Certificate III certificate_iii 1-2 years Trade qualification, skilled work, some autonomy
Certificate IV certificate_iv 1-2 years Supervisory roles, specialized skills, higher autonomy
Diploma diploma 1.5-2 years Management roles, complex tasks, leadership
Advanced Diploma advanced_diploma 2-3 years Senior management, strategic thinking, innovation
Vocational Graduate Certificate vocational_graduate_certificate 24-52 weeks Specialist skills for graduates
Vocational Graduate Diploma vocational_graduate_diploma 1-2 years Advanced specialist skills for graduates
Graduate Diploma graduate_diploma 1-2 years Professional development for degree holders

Structure Field

The structure JSON field contains the complete configuration for the template. Here's a breakdown of all possible fields:

Top-Level Structure Fields

Field Type Description
metadata Object Template version, author, purpose, target_audience, compliance_version
formatting Object Document style, fonts, colors, page layout
template_specific_settings Object Customization based on template type and AQF level
validation_rules Object Required sections, content length, compliance checks

Creating a Template

API Endpoint

POST /api/tenants/{tenant}/tas/templates/

Request Example

{
  "template_name": "Certificate IV Business",
  "template_code": "CERT_IV_BUS_2024",
  "template_type": "business",
  "aqf_level": "certificate_iv",
  "description": "For Certificate IV business qualifications with supervisory focus",
  "structure": {
    "metadata": {
      "version": "1.0",
      "author": "Business Team"
    },
    "formatting": {
      "document_style": "professional"
    },
    "template_specific_settings": {
      "emphasis_areas": {
        "practical_skills": 50,
        "theoretical_knowledge": 30,
        "supervisory_skills": 20
      }
    }
  },
  "is_active": true
}

Partial Update Example

‚ö†Ô∏è Important: Updates Don't Affect Existing TAS Documents

When you update a template, the changes only apply to new TAS documents generated after the update. Existing TAS documents retain the template configuration that was active when they were created.

Duplicating a Template

Use the duplicate action to create a copy of an existing template as a starting point:

API Endpoint

POST /api/tenants/{tenant}/tas/templates/{id}/duplicate/

Request Example

{
  "new_name": "Certificate IV Business - Regional Campus",
  "new_code": "CERT_IV_BUS_REGIONAL_2024"
}

This creates an exact copy with a new name and code, including all section assignments.

Best Practices

1. Use Descriptive Names and Codes

  • Include AQF level, industry sector, and year in the name
  • Use consistent naming conventions across your organization
  • Example: "Certificate III Hospitality - 2024"

2. Set Appropriate Emphasis Areas

  • Lower AQF levels: Higher practical emphasis (70-80%)
  • Higher AQF levels: More balanced or theory-heavy (40-60% each)
  • Trade qualifications: Always include safety as a category

3. Specify Realistic Delivery Preferences

  • Consider your RTO's actual delivery capacity
  • Match duration to AQF requirements
  • Include all delivery modes you actually offer

4. Document Resources Accurately

  • List all physical resources required for delivery
  • Include digital resources and their accessibility
  • Specify trainer qualifications needed

5. Keep Validation Rules Current

  • Review required sections annually
  • Update ASQA clause references when standards change
  • Adjust word counts based on actual document needs

Common Issues & Solutions

Issue Cause Solution
Template code already exists Duplicate template_code Use a unique code, include year/version
Generated content too generic Insufficient template_specific_settings Add detailed emphasis areas and assessment methods
Validation failing Missing required sections Check validation_rules.required_sections and assign all sections
Structure JSON too complex Over-configuration Start simple, add complexity as needed
