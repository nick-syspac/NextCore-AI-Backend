Scenario: ASQA audit requires adding a new section about industry consultation to all business templates.

Workflow

// 1. Create New Section
POST /api/tenants/acme-rto/tas/template-sections/
{
  "section_name": "Industry Consultation",
  "section_code": "INDUSTRY_CONSULT",
  "description": "How we engage with industry to ensure currency",
  "content_type": "rich_text",
  "is_editable": true,
  "gpt_prompt": {
    "system_prompt": "You are an expert in VET industry engagement and ASQA compliance...",
    "user_prompt_template": "Generate an industry consultation section for {qualification_code}. Include advisory committees, employer feedback, and currency validation processes. Address ASQA Clause 1.3.",
    "temperature": 0.6,
    "max_tokens": 800,
    "validation_keywords": ["industry", "consultation", "currency", "employers", "advisory"]
  }
}
// Response: { "id": 400, ... }

// 2. Get All Business Templates
GET /api/tenants/acme-rto/tas/templates/?template_type=business
// Response: List of 12 business templates

// 3. Add Section to Each Template (automation script)
const templates = [5, 8, 12, 15, 18, 21, 24, 27, 30, 33, 36, 39];

for (const templateId of templates) {
  // Get current max section_order
  const assignments = await GET(`/template-section-assignments/by_template/?template_id=${templateId}`);
  const maxOrder = Math.max(...assignments.map(a => a.section_order));
  
  // Add new section
  await POST('/template-section-assignments/', {
    template: templateId,
    section: 400,
    section_order: maxOrder + 1,
    is_required: true,
    custom_label: "Industry Consultation and Currency"
  });
}

// 4. Verify Addition
GET /api/tenants/acme-rto/tas/templates/5/
// Confirm new section appears in section_assignments array

Example 6: Batch Generation

Scenario: Generate TAS documents for 20 qualifications using the same template.

Script Approach

const qualifications = [
  { code: "BSB30120", title: "Certificate III in Business" },
  { code: "BSB40120", title: "Certificate IV in Business" },
  { code: "BSB50120", title: "Diploma of Business" },
  // ... 17 more
];

const templateId = 30; // Business template
const jobs = [];

// Initiate all generations
for (const qual of qualifications) {
  const response = await POST('/api/tenants/acme-rto/tas/generate/', {
    template_id: templateId,
    qualification_code: qual.code,
    qualification_title: qual.title,
    context: {
      target_audience: "working professionals",
      delivery_mode: "blended",
      duration: "calculated based on nominal hours"
    }
  });
  
  jobs.push({
    job_id: response.job_id,
    qual: qual.code
  });
}

// Monitor all jobs
const checkStatus = async () => {
  const statuses = await Promise.all(
    jobs.map(job => GET(`/api/tenants/acme-rto/tas/generation-status/${job.job_id}/`))
  );
  
  const completed = statuses.filter(s => s.status === 'completed').length;
  const failed = statuses.filter(s => s.status === 'failed').length;
  const inProgress = statuses.filter(s => s.status === 'generating').length;
  
  console.log(`Completed: ${completed}, Failed: ${failed}, In Progress: ${inProgress}`);
  
  if (inProgress === 0) {
    console.log('All generations complete!');
    return statuses;
  }
  
  // Check again in 15 seconds
  await sleep(15000);
  return checkStatus();
};

const results = await checkStatus();

// Download all completed TAS documents
for (const result of results.filter(r => r.status === 'completed')) {
  await GET(`/api/tenants/acme-rto/tas/${result.tas_id}/download/?format=pdf`);
}

Best Practices Summary

1. Plan Before Building

  • Map out all required sections
  • Identify sections that can be reused
  • Group similar qualifications under one template

2. Start Simple, Iterate

  • Begin with basic template and core sections
  • Generate sample TAS documents
  • Refine prompts based on output quality
  • Add complexity incrementally

3. Test Thoroughly

  • Use is_active=false during development
  • Generate multiple test TAS documents
  • Have compliance team review before production

4. Document Your Templates

  • Use meaningful names and codes
  • Fill in description fields
  • Document any template-specific quirks

5. Automate Repetitive Tasks

  • Script bulk section assignments
  • Automate batch TAS generation
  • Create maintenance scripts for updates

6. Version Control

  • Include year in template codes
  • Duplicate before major changes
  • Keep old templates inactive for reference

Common Patterns

Pattern When to Use Implementation
Master Template + Duplicates Multiple similar configurations Create master, duplicate for variations
Section Library First Building from scratch Create all sections, then templates
Template-Level Prompts AQF-specific language Override section prompts in template.gpt_prompts
Staged Rollout Large changes Start inactive, test, then activate
Batch Processing Multiple TAS needed Script to loop through qualifications

Troubleshooting Common Issues

Issue Likely Cause Solution
Generated content too generic GPT prompt lacks specificity Add more context variables, refine user_prompt_template
Content inconsistent across sections Temperature too high Lower temperature to 0.5-0.6
Section content cut off max_tokens too low Increase max_tokens by 50%
Validation failing Keywords don't appear naturally Adjust keywords or add explicit requirement in prompt
Generation takes too long Too many sections, high max_tokens Split into batches, optimize max_tokens
Wrong section order in document section_order values incorrect Use reorder endpoint to fix
