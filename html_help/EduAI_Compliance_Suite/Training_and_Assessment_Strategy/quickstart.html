Step 1: Create Reusable Sections

First, create sections that can be reused across multiple templates. These form your section library.

Example: Create a Cohort Profile Section

POST /api/tenants/{tenant_slug}/tas/template-sections/

Request Body:

{
  "section_name": "Cohort Profile and Entry Requirements",
  "section_code": "cohort_profile",
  "description": "Profile of target learners including demographics, LLN levels, and entry requirements",
  "content_type": "rich_text",
  "is_editable": true,
  "default_content": "This section describes the target student cohort...",
  "gpt_prompt": {
    "system_prompt": "You are an expert VET educator specializing in Australian RTO compliance and ASQA standards.",
    "user_prompt_template": "Create a comprehensive cohort profile for {qualification_code} - {qualification_title}, a {aqf_level} qualification in {industry_sector}. Include: target demographics, entry requirements, LLN considerations, and support needs.",
    "temperature": 0.7,
    "max_tokens": 1500,
    "variables": ["qualification_code", "qualification_title", "aqf_level", "industry_sector"],
    "validation_keywords": ["demographics", "LLN", "entry requirements"]
  }
}

Response:

{
  "id": 1,
  "section_name": "Cohort Profile and Entry Requirements",
  "section_code": "cohort_profile",
  "content_type": "rich_text",
  "is_active": true,
  "created_at": "2024-11-03T10:00:00Z"
}

üí° Pro Tip

Create multiple sections at once for your template. Common sections include: cohort_profile, lln_support, delivery_model, assessment_overview, unit_clustering, trainer_requirements, and quality_assurance.

Step 2: Create a TAS Template

Define a template with structure configuration and GPT prompts.

POST /api/tenants/{tenant_slug}/tas/templates/

Request Body:

{
  "template_name": "Certificate I - General Foundation Skills",
  "template_code": "CERT1_GENERAL_001",
  "aqf_level": "certificate_i",
  "template_type": "general",
  "description": "Foundation skills template for entry-level qualifications",
  "structure": {
    "version": "1.0",
    "template_type": "general",
    "aqf_level": "certificate_i",
    "metadata": {
      "author_guidelines": "Focus on basic foundational skills. Use plain language.",
      "compliance_framework": "ASQA Standards 2015",
      "industry_focus": "Multi-sector introductory skills"
    },
    "formatting": {
      "font": "Arial",
      "font_size": 12,
      "line_spacing": 1.5
    },
    "validation_rules": {
      "required_sections": ["cohort_profile", "delivery_model", "assessment_overview"],
      "min_length": 3000
    },
    "template_specific_settings": {
      "emphasis_areas": [
        "basic_foundational_skills",
        "workplace_readiness",
        "high_support"
      ],
      "assessment_methods": [
        "simple_practical_demonstrations",
        "observation_checklists",
        "verbal_questioning"
      ]
    }
  },
  "is_active": true
}

Response:

{
  "id": 5,
  "template_name": "Certificate I - General Foundation Skills",
  "template_code": "CERT1_GENERAL_001",
  "aqf_level": "certificate_i",
  "template_type": "general",
  "created_at": "2024-11-03T10:05:00Z"
}

The structure field is a comprehensive JSON configuration. See Template Details for complete structure examples.

Step 4: Generate TAS Content

Use AI to generate content for individual sections or the entire TAS document.

Option A: Generate Single Section

POST /api/tenants/{tenant_slug}/tas/templates/{template_id}/generate-section/

Request Body:

{
  "section_code": "cohort_profile",
  "context": {
    "qualification_code": "FSK10119",
    "qualification_title": "Certificate I in Foundation Skills",
    "aqf_level": "certificate_i",
    "industry_sector": "Foundation Skills",
    "nominal_hours": 200,
    "unit_count": 8
  }
}

Response:

{
  "section_code": "cohort_profile",
  "section_name": "Foundation Skills Cohort Profile",
  "content": "The Certificate I in Foundation Skills (FSK10119) targets learners who are new to formal education or training...",
  "word_count": 1245,
  "generation_time_ms": 3421,
  "model_used": "gpt-4",
  "validation": {
    "passed": true,
    "keywords_found": ["demographics", "LLN", "entry requirements"],
    "missing_keywords": []
  }
}

Common Workflows

Creating Multiple Qualifications from Same Template

  1. Create template and section library once
  2. For each qualification:
    • Call generate endpoint with qualification-specific context
    • Review and customize content
    • Approve and publish

Updating Existing TAS

  1. Fetch existing TAS document
  2. Regenerate specific sections that need updates
  3. Merge with existing content
  4. Create new version

Creating Custom Template

  1. Start with closest matching template type
  2. Duplicate template
  3. Modify structure and GPT prompts
  4. Test with sample qualification
  5. Refine based on output quality

Troubleshooting

Issue Solution
Section validation fails Check that required validation keywords are mentioned in GPT prompt and adjust temperature
Content too generic Add more specific context variables and refine system prompt for that section
Generation times out Reduce max_tokens in GPT prompts or generate sections individually
Inconsistent tone Ensure all section prompts have consistent tone guidelines in system prompt
