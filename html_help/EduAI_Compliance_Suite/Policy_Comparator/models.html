Foundation Phase Models

ğŸ“š ASQAStandard

Purpose: Repository of ASQA standards (Standards for RTOs 2015)

Database Table: asqa_standards

Fields

Field Name Type Constraints Description
id AutoField Primary Key Auto-incrementing ID
standard_number CharField(20) unique, indexed Standard identifier (e.g., "1.1", "2.1")
title CharField(300) required Standard title
description TextField required Standard description/overview
standard_type CharField(50) choices Category (training_assessment, trainer_assessor, educational_support, certification, engagement_employer, complaints_appeals, governance, financial)
full_text TextField required Complete text of the ASQA standard
requirements JSONField default=[] List of specific requirements
is_active BooleanField default=True Whether standard is currently active
effective_date DateField nullable When standard became effective
version CharField(20) default="2015" Standard version
created_at DateTimeField auto_now_add Record creation timestamp
updated_at DateTimeField auto_now Last update timestamp

Relationships

  • clauses: One-to-Many with ASQAClause (reverse relation)

Meta Options

  • Ordering: ["standard_number"]
  • Verbose Name: "ASQA Standard"

ğŸ“„ ASQAClause

Purpose: Individual clauses within ASQA standards for granular comparison

Database Table: asqa_clauses

Fields

Field Name Type Constraints Description
id AutoField Primary Key Auto-incrementing ID
standard ForeignKey CASCADE, indexed Parent ASQAStandard
clause_number CharField(20) required Clause identifier (e.g., "1.1", "1.2")
title CharField(300) required Clause title
clause_text TextField required Full text of the clause for NLP comparison
evidence_required JSONField default=[] Types of evidence needed for compliance
keywords JSONField default=[] Key terms for text similarity matching
compliance_level CharField(20) choices critical, essential, or recommended
is_active BooleanField default=True Whether clause is currently active
created_at DateTimeField auto_now_add Record creation timestamp
updated_at DateTimeField auto_now Last update timestamp

Relationships

  • standard: Many-to-One with ASQAStandard
  • comparison_results: One-to-Many with ComparisonResult

Constraints

  • Unique Together: ["standard", "clause_number"]

Indexes

  • Foreign key: standard_id
  • Composite: (standard, clause_number)

Purpose: Organisation policies to be compared against ASQA standards

Database Table: policies

Fields

Field Name Type Constraints Description
id AutoField Primary Key Auto-incrementing ID
tenant ForeignKey CASCADE, indexed Multi-tenant isolation
policy_number CharField(50) required Policy identifier (e.g., "TAS-001")
title CharField(300) required Policy title
description TextField blank=True Policy description/overview
policy_type CharField(50) choices training_delivery, assessment, student_support, complaints_appeals, financial, governance, quality_assurance, workplace_health_safety, staff_management, facilities_resources
content TextField required Full policy content for comparison
version CharField(20) default="1.0" Policy version number
status CharField(20) choices, default="draft" draft, under_review, approved, archived
effective_date DateField nullable When policy becomes effective
review_date DateField nullable Next scheduled review date
last_compared_at DateTimeField nullable Most recent comparison timestamp
compliance_score FloatField nullable, 0-100 Overall compliance score percentage
file_path CharField(500) blank=True Path to uploaded policy document
created_by ForeignKey(User) SET_NULL, nullable User who created policy
created_at DateTimeField auto_now_add Record creation timestamp
updated_at DateTimeField auto_now Last update timestamp

Relationships

  • tenant: Many-to-One with Tenant
  • created_by: Many-to-One with User
  • comparison_results: One-to-Many with ComparisonResult
  • comparison_sessions: One-to-Many with ComparisonSession

Constraints

  • Unique Together: ["tenant", "policy_number"]

Indexes

  • Foreign key: tenant_id
  • Composite: (tenant, status)
  • Single: policy_type
  • Single: compliance_score

ğŸ“Š ComparisonResult

Purpose: Detailed NLP-based comparison results for each policy-clause pair

Database Table: comparison_results

Fields

Field Name Type Constraints Description
id AutoField Primary Key Auto-incrementing ID
policy ForeignKey CASCADE, indexed Policy being compared
asqa_clause ForeignKey CASCADE, indexed ASQA clause being compared against
similarity_score FloatField 0.0-1.0 NLP text similarity score
match_type CharField(20) choices full, partial, weak, no_match (auto-set)
matched_text TextField blank=True Relevant text excerpt from policy
gap_description TextField blank=True Description of compliance gap
recommendations JSONField default=[] Suggestions to improve compliance
nlp_metadata JSONField default={} NLP processing metadata (keyword_score, sequence_score, phrase_score, method)
keywords_matched JSONField default=[] Keywords found in policy
keywords_missing JSONField default=[] Keywords missing from policy
has_sufficient_evidence BooleanField default=False Whether evidence is sufficient
evidence_notes TextField blank=True Notes about evidence sufficiency
is_compliant BooleanField default=False Meets ASQA requirements
requires_action BooleanField default=False Action needed to achieve compliance
comparison_date DateTimeField auto_now_add When comparison was performed
reviewed_by ForeignKey(User) SET_NULL, nullable User who reviewed result
reviewed_at DateTimeField nullable When result was reviewed

Relationships

  • policy: Many-to-One with Policy
  • asqa_clause: Many-to-One with ASQAClause
  • reviewed_by: Many-to-One with User

Constraints

  • Unique Together: ["policy", "asqa_clause"]

Indexes

  • Composite: (policy, similarity_score)
  • Single: is_compliant
  • Single: requires_action

Auto-Save Logic

def save(self, *args, **kwargs): # Auto-classify match type based on similarity score if self.similarity_score >= 0.8: self.match_type = "full" self.is_compliant = True elif self.similarity_score >= 0.6: self.match_type = "partial" self.requires_action = True elif self.similarity_score >= 0.4: self.match_type = "weak" self.requires_action = True else: self.match_type = "no_match" self.requires_action = True super().save(*args, **kwargs)

ğŸ”„ ComparisonSession

Purpose: Track batch comparison sessions for historical analysis

Database Table: comparison_sessions

Fields

Field Name Type Constraints Description
id AutoField Primary Key Auto-incrementing ID
tenant ForeignKey CASCADE, indexed Tenant isolation
policy ForeignKey CASCADE, indexed Policy being compared
session_name CharField(200) required User-defined session identifier
status CharField(20) choices, default="pending" pending, processing, completed, failed
standards_compared JSONField default=[] List of ASQA standard IDs compared
total_clauses_checked IntegerField default=0 Number of clauses analyzed
compliant_count IntegerField default=0 Full matches count
partial_match_count IntegerField default=0 Partial matches count
gap_count IntegerField default=0 Weak/no matches count
overall_compliance_score FloatField nullable, 0-100 Weighted compliance score
processing_time_seconds FloatField default=0.0 Comparison execution time
error_message TextField blank=True Error details if status=failed
created_by ForeignKey(User) SET_NULL, nullable User who initiated session
created_at DateTimeField auto_now_add Session start timestamp
completed_at DateTimeField nullable Session completion timestamp

Relationships

  • tenant: Many-to-One with Tenant
  • policy: Many-to-One with Policy
  • created_by: Many-to-One with User

Planned Models (Phase 2+)

ğŸ“„ Document

Purpose: Uploaded policy documents with versioning support (PDF/DOCX)

Status: Planned for Phase 2

Database Table: policy_documents

Key Features

  • S3 storage for document files
  • SHA256 hash for deduplication
  • Multi-format support (PDF, DOCX, HTML, TXT)
  • Status tracking (pending, uploading, processing, ready, failed, archived)
  • Metadata: pages, word_count, language

ğŸ“ DocumentVersion

Purpose: Version history for documents

Status: Planned for Phase 2

Database Table: document_versions

Key Features

  • Track document revisions over time
  • Maintain S3 URIs for each version
  • SHA256 hash per version for integrity
  • Sequential version numbering

ğŸ§© DocumentChunk

Purpose: Text chunks from documents for embedding and comparison

Status: Planned for Phase 2

Database Table: document_chunks

Key Features

  • Chunking strategy: 300-800 tokens with overlap
  • Section path tracking (heading hierarchy)
  • Page range tracking (page_from, page_to)
  • Token count for each chunk
  • Prepared for vector embeddings (Phase 3)

ğŸ“¦ AsqaClausePack

Purpose: Versioned packs of ASQA clauses (global or tenant-specific)

Status: Planned for Phase 2

Database Table: asqa_clause_packs

Key Features

  • Versioned clause collections
  • Global packs (tenant=NULL) or tenant-specific
  • Status workflow: draft, review, published, archived
  • Enables custom clause sets per RTO
  • Published date tracking
