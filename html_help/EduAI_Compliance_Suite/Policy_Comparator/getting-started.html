Prerequisites

What You'll Need

  • API Access: Valid tenant credentials and API endpoint
  • Authentication: API key or OAuth token
  • Policy Documents: Your organization's policies (text or documents)
  • ASQA Standards: Pre-loaded or custom clause packs

System Requirements

  • HTTP client (cURL, Postman, or programming language with HTTP library)
  • JSON parsing capability
  • Text editor for policy content preparation

Create a Policy

Step 1: Prepare Policy Content

Ensure your policy content is ready in plain text format. Key points:

  • Include complete policy text (not just excerpts)
  • Remove excessive formatting that might interfere with NLP analysis
  • Ensure critical compliance keywords are present
  • Use clear, professional language

Step 2: Create Policy via API

POST /api/tenants/{tenant_slug}/policy-comparator/policies/ 
curl -X POST \
  'https://api.yourorg.com/api/tenants/your-tenant/policy-comparator/policies/' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
  "policy_number": "POL-001",
  "title": "Trainer and Assessor Policy",
  "description": "Policy outlining requirements for trainer and assessor qualifications",
  "policy_type": "training_delivery",
  "content": "Purpose: This policy outlines the requirements for trainer and assessor qualifications...\n\nScope: This policy applies to all trainers and assessors delivering training and assessment...\n\nRequirements:\n1. All trainers must hold relevant qualifications\n2. Trainers must demonstrate current industry skills\n3. Regular professional development is required...",
  "version": "1.0",
  "status": "approved",
  "effective_date": "2025-01-01",
  "review_date": "2026-01-01"
}'

Response:
{
  "id": 42,
  "policy_number": "POL-001",

Understanding the Results

Similarity Score (0-1)

Indicates how well your policy matches each ASQA clause:

  • 0.8-1.0: Excellent coverage (Full Match)
  • 0.6-0.8: Good coverage with minor gaps (Partial Match)
  • 0.4-0.6: Weak coverage with significant gaps (Weak Match)
  • <0.4: Inadequate coverage (No Match)

Match Type Categories

Match Type Meaning Action
full_match Compliant - no action needed ‚úì Continue monitoring
partial_match Review recommended ‚ö† Consider enhancements
weak_match Action required ‚ö° Update policy
no_match Critical gap üö® Immediate attention

NLP Metadata

Three scores combine to produce the final similarity score:

  • keyword_score: Presence of compliance keywords (40% weight)
  • sequence_score: Text structure similarity (30% weight)
  • phrase_score: Semantic meaning alignment (30% weight)

Prioritizing Gap Resolution

Focus on gaps in this order:

  1. Critical Gaps: ASQA clauses marked as "critical" with no_match or weak_match
  2. Moderate Gaps: Important clauses with partial or weak matches
  3. Minor Gaps: Standard clauses needing minor improvements
‚ö†Ô∏è Audit Risk: Critical gaps in compliance-level clauses pose the highest audit risk and should be addressed immediately.

Batch Comparison

Compare Multiple Policies

For comprehensive compliance reviews, compare multiple policies in a single session:

# Compare all training-related policies
for policy_id in [42, 43, 44, 45]; do
  curl -X POST \
    "https://api.yourorg.com/api/tenants/your-tenant/policy-comparator/policies/${policy_id}/compare/" \
    -H 'Authorization: Bearer YOUR_TOKEN' \
    -H 'Content-Type: application/json' \
    -d '{
      "standards": ["1.1", "1.2", "1.3", "1.8"],
      "session_name": "Training Policy Suite Review"
    }'
done
