Prerequisites

What You'll Need

  • API Access: Valid tenant credentials and API endpoint
  • Authentication: API key or OAuth token
  • Qualification Information: Training.gov.au code for the qualification
  • Context Data: Delivery mode, duration, target audience details

System Requirements

  • HTTP client (cURL, Postman, or programming language with HTTP library)
  • JSON parsing capability
  • Modern web browser for documentation

Authentication

API Authentication

All API requests require authentication via bearer token:

Authorization: Bearer YOUR_API_TOKEN

Getting Your Token

POST /api/auth/token/
Content-Type: application/json

{
  "username": "your_username",
  "password": "your_password"
}

Response:
{
  "access": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "refresh": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}

Create Your First TAS

Step 1: List Available Qualifications

First, retrieve the list of qualifications you can create TAS documents for:

GET /api/tenants/{tenant_slug}/tas/qualifications/ 
curl -X GET \
  'https://api.yourorg.com/api/tenants/your-tenant/tas/qualifications/' \
  -H 'Authorization: Bearer YOUR_TOKEN'

Response:

  "qualifications": [

      "code": "BSB50120",
      "title": "Diploma of Business",

Step 3: Generate TAS Document

Create a complete TAS document with AI-powered generation:

POST /api/tenants/{tenant_slug}/tas/generate/ 
curl -X POST \
  'https://api.yourorg.com/api/tenants/your-tenant/tas/generate/' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
  "code": "BSB50120",
  "qualification_name": "Diploma of Business",
  "aqf_level": "AQF_5",
  "training_package": "BSB - Business Services",
  "delivery_mode": "blended",
  "duration_weeks": 52,
  "target_audience": "working professionals, career changers",
  "units_of_competency": [
    "BSBCRT511",
    "BSBFIN501",
    "BSBOPS502"
  ],
  "assessment_methods": [
    "Written assessments",
    "Practical demonstrations",
    "Projects",
    "Portfolio of evidence"
  ],
  "use_gpt4": true,
  "additional_context": "Focus on practical workplace application"
}'

Response:
{
  "id": 42,

  "code": "BSB50120",
  "status": "generating",
  "message": "TAS generation started successfully",

}

Working with Templates

What are TAS Templates?

Templates define the structure, sections, and AI prompts used for TAS generation. They can be customized for different:

  • AQF Levels (Certificate I-IV, Diploma, Advanced Diploma)
  • Training packages
  • Delivery modes
  • Organizational preferences

List Available Templates

GET /api/tenants/{tenant_slug}/tas/templates/ 
Response:
[
  {
    "id": 5,
    "name": "Certificate III Standard Template",
    "template_code": "CERT3_STANDARD",
    "aqf_level": "AQF_3",
    "is_active": true,
    "sections_count": 10
  },
  {
    "id": 6,
    "name": "Diploma Business Template",
    "template_code": "DIPLOMA_BUS",
    "aqf_level": "AQF_5",
    "is_active": true,
    "sections_count": 12
  }
]

Create Custom Template

POST /api/tenants/{tenant_slug}/tas/templates/ 
{
  "name": "Custom Certificate IV Template",
  "template_code": "CERT4_CUSTOM",
  "aqf_level": "AQF_4",
  "description": "Customized template for Certificate IV programs",
  "default_sections": [
    "qualification_overview",
    "target_group",

    "qualification_overview": {
      "temperature": 0.7,
      "max_tokens": 1000,
      "system_message": "You are an expert in Australian VET..."
    }
  }
}

Using AI Features

AI-Powered Enhancements

The TAS Generator includes 9 major AI feature areas. Here are the most commonly used:

1. Enrich TGA Snapshot

Automatically enhance Training.gov.au data with learning outcomes and assessment hints:

POST /api/tenants/{tenant_slug}/tas/{id}/ai/enrich-tga/ 
{
  "unit_code": "BSBWHS411",
  "tga_data": {
    "title": "Implement and monitor WHS policies",
    "elements": [...],
    "performance_criteria": [...]
  }
}

Response:
{
  "learning_outcomes": [
    "Implement WHS consultation processes",
    "Monitor WHS performance indicators"
  ],
  "assessment_hints": [
    "Practical demonstration of consultation",
    "Case study analysis"
  ],
  "industry_contexts": ["Construction", "Manufacturing"],
  "delivery_notes": "Focus on practical application"
}

2. Recommend Electives

Get AI-powered elective recommendations based on job outcomes:

POST /api/tenants/{tenant_slug}/tas/{id}/ai/recommend-electives/ 
{
  "qualification_code": "BSB50120",
  "job_outcomes": ["Business Manager", "Operations Coordinator"],
  "available_electives": [...]
}

Response:
[
  {
    "unit_code": "BSBOPS502",
    "relevance_score": 0.95,
    "rationale": "Critical for Business Manager role",
    "skills_developed": ["Operational planning", "Resource allocation"]
  }
]

3. Generate Assessment Blueprint

Create comprehensive assessment plans for units:

POST /api/tenants/{tenant_slug}/tas/{id}/ai/assessment-blueprint/ 
{
  "unit_code": "BSBWHS411",
  "elements": [...],
  "delivery_mode": "workplace",
  "industry_context": "construction"
}

Response:
{
  "assessment_tasks": [
    {
      "task_number": 1,
      "name": "WHS Implementation Project",
      "elements_assessed": ["Element 1", "Element 2"],
      "instrument_type": "project"
    }
  ]
}

Editing & Refinement

Update TAS Content

Modify generated content as needed:

PATCH /api/tenants/{tenant_slug}/tas/{id}/ 
{
  "sections": [
    {
      "key": "qualification_overview",
      "content": "
Updated Overview
Modified content...
"
    }
  ],
  "status": "review"
}

Regenerate Specific Section

If you're not happy with a section, regenerate it:

POST /api/tenants/{tenant_slug}/tas/{id}/regenerate_section/ 
{
  "section_key": "learning_resources"
}

Response:
{
  "section_key": "learning_resources",
  "content": "
Learning Resources
Regenerated content...
",
  "tokens_used": 420
}

Compliance Checking

Run Compliance Validation

Check your TAS against ASQA Standards:

POST /api/tenants/{tenant_slug}/tas/{id}/ai/check-compliance/ 
{
  "tas_content": {...},
  "target_clauses": ["1.1", "1.2", "1.3", "1.8"]
}

Response:
{
  "overall_score": 0.85,
  "clause_scores": {
    "1.1": {
      "score": 0.90,
      "status": "compliant",
      "evidence": ["Section 2.1", "Section 3.4"],
      "gaps": []
    },
    "1.2": {
      "score": 0.75,
      "status": "partial",
      "gaps": ["Missing delivery strategy detail"]
    }
  },
  "recommendations": [
    "Add more detail to assessment validation process",
    "Include specific trainer requirements for each unit"
  ]
}

Export & Finalize

Export to PDF

Generate a professional PDF document:

POST /api/tenants/{tenant_slug}/tas/{id}/export/pdf/ 
{
  "include_cover_page": true,
  "include_toc": true,
  "branding": {
    "logo_url": "https://yourorg.com/logo.png",
    "primary_color": "#2563eb"
  }
}

Response:
{
  "download_url": "https://api.yourorg.com/downloads/tas-42.pdf",
  "expires_at": "2024-11-07T10:30:00Z"
}

Approve and Version

Mark TAS as approved and create a version snapshot:

POST /api/tenants/{tenant_slug}/tas/{id}/approve/ 
{
  "status": "approved",
  "approved_by": "Jane Smith",
  "approval_date": "2024-11-06",
  "notes": "Approved for 2025 delivery"
}

Response:
{
  "id": 42,
  "status": "approved",
  "version": 1,
  "is_current_version": true,
  "approved_at": "2024-11-06T10:30:00Z"
}
