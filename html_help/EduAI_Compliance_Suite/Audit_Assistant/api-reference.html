Authentication

All API requests require authentication using JWT tokens in the Authorization header:

Authorization: Bearer <your_jwt_token>

Base URL: /api/v1/tenants/{tenant_id}/audit-assistant/

Evidence Management API

List Evidence

Endpoint: GET /evidence/

Retrieve paginated list of evidence documents for the tenant.

Query Parameters

Parameter Type Description
status string Filter by status: uploaded, processing, tagged, reviewed, approved, rejected
evidence_type string Filter by type: policy, procedure, record, etc.
search string Search in title, description, evidence_number
ordering string Sort field: uploaded_at, title, evidence_date
page integer Page number (default: 1)
page_size integer Items per page (default: 20, max: 100)

Response Example

{ "count": 45, "next": "/api/v1/tenants/123/audit-assistant/evidence/?page=2", "previous": null, "results": [ { "id": 1, "evidence_number": "EV-2024-001", "title": "Assessment Policy", "description": "Comprehensive assessment policy document", "evidence_type": "policy", "file": "/media/evidence/123/assessment-policy.pdf", "file_url": "https://cdn.example.com/evidence/123/assessment-policy.pdf", "file_size": 245760, "status": "tagged", "tags": ["assessment", "policy", "2024"], "evidence_date": "2024-01-15", "uploaded_at": "2024-01-16T10:30:00Z", "uploaded_by_name": "Jane Doe", "ner_processed_at": "2024-01-16T10:30:05Z", "tagged_clauses_count": 8, "reviewed_at": null, "reviewed_by": null } ] }

Upload Evidence

Endpoint: POST /evidence/upload/

Upload evidence document with automatic NER processing.

Request Headers

Content-Type: multipart/form-data

Request Body (Form Data)

Field Type Required Description
file file Yes Evidence file (PDF, DOCX, TXT, etc.)
evidence_number string Yes Unique evidence identifier
title string Yes Evidence title
description string No Detailed description
evidence_type string Yes Type: policy, procedure, record, etc.
evidence_date date Yes Date evidence was created (YYYY-MM-DD)
tags array No Custom tags (JSON array)

Python Example

import requests url = "https://api.example.com/api/v1/tenants/123/audit-assistant/evidence/upload/" headers = { "Authorization": "Bearer your_jwt_token" } files = { "file": open("assessment-policy.pdf", "rb") } data = { "evidence_number": "EV-2024-001", "title": "Assessment Policy", "description": "Comprehensive assessment policy document", "evidence_type": "policy", "evidence_date": "2024-01-15", "tags": '["assessment", "policy", "2024"]' } response = requests.post(url, headers=headers, files=files, data=data) print(response.json())

Response (201 Created)

{ "id": 1, "evidence_number": "EV-2024-001", "title": "Assessment Policy", "status": "processing", "message": "Evidence uploaded successfully. NER processing started.", "task_id": "a8b7c6d5-e4f3-2g1h-0i9j-8k7l6m5n4o3p" }

Retrieve auto-tagged clauses for an evidence document with confidence scores.

Verify Clause Mapping

Endpoint: POST /evidence/{id}/verify_tagging/

Verify or reject auto-tagged clause mappings.

Request Body

{ "clause_evidence_ids": [45, 46, 47], "action": "verify" // or "reject" }

Response (200 OK)

{ "message": "3 clause mappings verified successfully", "verified_count": 3, "rejected_count": 0 }

Process NER Manually

Endpoint: POST /evidence/{id}/process_ner/

Trigger NER processing manually (if auto-processing failed).

Response (200 OK)

{ "message": "NER processing started", "task_id": "a8b7c6d5-e4f3-2g1h-0i9j-8k7l6m5n4o3p" }

Update Evidence

Endpoint: PUT /evidence/{id}/ or PATCH /evidence/{id}/

Update evidence metadata (not the file itself).

Request Body

{ "title": "Updated Assessment Policy", "description": "Updated description", "tags": ["assessment", "policy", "2024", "updated"], "status": "reviewed" }

Delete Evidence

Endpoint: DELETE /evidence/{id}/

Delete evidence document and all associated clause mappings.

Response (204 No Content)

Clause Evidence API

List Clause Evidence

Endpoint: GET /clause-evidence/

Retrieve clause-to-evidence mappings with filtering options.

Query Parameters

Parameter Type Description
mapping_type string Filter: auto_ner, auto_rule, manual, suggested
is_verified boolean Filter by verification status
min_confidence float Minimum confidence score (0.0-1.0)
asqa_clause integer Filter by specific clause ID
evidence integer Filter by specific evidence ID

Identify Gaps

Endpoint: GET /clause-evidence/gaps/

Identify ASQA clauses without evidence (compliance gaps).

Query Parameters

Parameter Type Description
standard_ids array Comma-separated standard IDs (e.g., "1,2,3")
severity string Filter by severity: critical, major, minor

Response Example

{ "total_clauses": 150, "clauses_with_evidence": 120, "clauses_without_evidence": 30, "coverage_percentage": 80.0, "gaps": [ { "clause": { "id": 25, "clause_number": "1.5.3", "title": "Marketing and advertising compliance", "standard_number": "1.5", "is_critical": true }, "evidence_count": 0, "verified_evidence_count": 0, "severity": "critical", "recommendation": "Upload marketing compliance policy and recent advertising materials" }, { "clause": { "id": 42, "clause_number": "2.3.1", "title": "Trainer qualifications verification", "standard_number": "2.3", "is_critical": false }, "evidence_count": 0, "verified_evidence_count": 0, "severity": "major", "recommendation": "Provide trainer qualification certificates and verification records" } ] }

Create Manual Mapping

Endpoint: POST /clause-evidence/

Create manual clause-to-evidence mapping.

Request Body

{ "asqa_clause": 25, "evidence": 15, "mapping_type": "manual", "relevance_notes": "This policy directly addresses clause requirements" }

Delete Mapping

Endpoint: DELETE /clause-evidence/{id}/

Remove clause-to-evidence mapping.

Submit Report

Pagination

All list endpoints support cursor-based pagination:

{ "count": 245, "next": "/api/v1/tenants/123/audit-assistant/evidence/?page=3", "previous": "/api/v1/tenants/123/audit-assistant/evidence/?page=1", "results": [...] }

Error Handling

HTTP Status Codes

Status Code Meaning Description
200 OK Request succeeded
201 Created Resource created successfully
204 No Content Resource deleted successfully
400 Bad Request Invalid request data
401 Unauthorized Missing or invalid authentication
403 Forbidden Insufficient permissions
404 Not Found Resource does not exist
409 Conflict Duplicate resource (e.g., evidence_number)
413 Payload Too Large File exceeds maximum size
500 Internal Server Error Server error occurred

Error Response Format

{ "error": "Validation Error", "detail": "Evidence with this number already exists", "field": "evidence_number", "code": "unique" }
