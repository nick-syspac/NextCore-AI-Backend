1. ImprovementAction

Main entity tracking individual improvement actions with AI classification and compliance features.

Field Type Description
id Integer (PK) Auto-incrementing primary key
tenant ForeignKey Tenant isolation (required)
action_number CharField(50) Unique identifier (e.g., CIR-2025-0123)
title CharField(255) Action title (required)
description TextField Detailed description
category ForeignKey ImprovementCategory (nullable, AI-set)
priority CharField(20) Choices: critical, high, medium, low
status CharField(20) Choices: identified, planned, in_progress, on_hold, completed, cancelled
source CharField(30) 11 source types (audit, complaint, feedback, etc.)
risk_level CharField(20) AI-assessed: critical, high, medium, low (nullable)
root_cause TextField Root cause analysis (optional)
expected_impact TextField Expected outcome description
actual_impact TextField Post-implementation results
effectiveness_rating Integer 1-5 rating after completion (nullable)
compliance_status CharField(20) Choices: compliant, at_risk, overdue, completed
is_critical_compliance Boolean Affects RTO registration (default: False)
requires_approval Boolean Needs management approval (default: False)
approved_at DateTimeField Approval timestamp (nullable)
tags JSONField Custom tags array (default: [])
ai_summary TextField AI-generated summary (optional)
ai_classification_metadata JSONField Classification confidence, keywords, reasoning
identified_by ForeignKey(User) User who created action
assigned_to ForeignKey(User) Current owner (nullable)
approved_by ForeignKey(User) Approver (nullable)
target_date DateField Target completion date (nullable)
due_date DateField SLA-based due date (nullable)
started_at DateTimeField When work began (nullable)
completed_at DateTimeField Completion timestamp (nullable)
created_at DateTimeField Record creation (auto)
updated_at DateTimeField Last update (auto)

Properties & Methods

  • is_overdue - Boolean property checking if past due_date
  • days_since_creation - Integer days since created
  • time_to_completion - Timedelta between created and completed
  • save() - Generates action_number if not set

2. ImprovementCategory

ASQA-aligned categories for organizing improvement actions.

Field Type Description
id Integer (PK) Primary key
tenant ForeignKey Tenant isolation
name CharField(200) Category name
category_type CharField(50) 12 predefined types (training_assessment, student_support, etc.)
description TextField Category description
color_code CharField(7) Hex color for UI (default: #3B82F6)
related_standards JSONField ASQA standard numbers array
is_active Boolean Active status (default: True)
created_at DateTimeField Creation timestamp
created_by ForeignKey(User) Creator (nullable)

3. ActionStep

Individual workflow steps within an improvement action for detailed tracking.

Field Type Description
id Integer (PK) Primary key
improvement_action ForeignKey Parent action (required)
sequence_order Integer Step order (10, 20, 30...)
title CharField(255) Step title
description TextField Step details
status CharField(20) Choices: not_started, in_progress, completed, cancelled
owner ForeignKey(User) Step owner (nullable)
due_date DateField Step due date (nullable)
started_at DateTimeField When started (nullable)
completed_at DateTimeField Completion time (nullable)
progress_notes TextField Progress documentation
evidence_refs JSONField Evidence references array
is_blocked Boolean Blocker flag (default: False)
blocker_description TextField Blocker details
blocker_resolved_at DateTimeField When blocker resolved (nullable)
created_at DateTimeField Creation timestamp
updated_at DateTimeField Last update

Properties

  • is_overdue - Boolean: True if past due_date and not completed/cancelled

4. Comment

Threaded comments on improvement actions with @mention support.

Field Type Description
id Integer (PK) Primary key
improvement_action ForeignKey Parent action
author ForeignKey(User) Comment author
body TextField Comment text (supports Markdown)
parent_comment ForeignKey(self) Parent for threaded replies (nullable)
mentions ManyToManyField Mentioned users
is_internal Boolean Internal note vs public (default: False)
created_at DateTimeField Creation timestamp
updated_at DateTimeField Last edit timestamp

5. Attachment

File attachments with integrity hashing and metadata.

Field Type Description
id Integer (PK) Primary key
improvement_action ForeignKey Parent action
file FileField Uploaded file
filename CharField(255) Original filename
file_size BigInteger Size in bytes
content_type CharField(100) MIME type
hash_sha256 CharField(64) SHA-256 integrity hash
kind CharField(50) 8 types: evidence, analysis, report, approval, photo, correspondence, policy, other
description TextField File description
uploaded_by ForeignKey(User) Uploader
uploaded_at DateTimeField Upload timestamp

Links between improvement actions and ASQA/ISO clauses.
