üîÑ Common Workflows

Step-by-step guides for common CIR workflows from initial identification through verification and closure.

1. üìù Audit Finding Response

Complete workflow for addressing audit findings.

Step 1: Create Action from Finding

POST /api/v1/continuous_improvement/actions/
{
  "title": "Address Standard 1.8 evidence gap",
  "description": "Auditor identified insufficient evidence of systematic monitoring. Implement quarterly review process with documented outcomes.",
  "source": "audit",
  "priority": "critical",
  "assigned_to_id": 12
}

Step 2: AI Classification

POST /api/v1/continuous_improvement/actions/123/classify/

Result: Category set to "compliance_governance", risk level "critical", linked to clause 1.8

Step 3: Create Corrective Steps

POST /api/v1/continuous_improvement/steps/
{
  "improvement_action_id": 123,
  "sequence_order": 10,
  "title": "Design quarterly review template",
  "owner_id": 12,
  "due_date": "2025-11-15"
}

POST /api/v1/continuous_improvement/steps/
{
  "improvement_action_id": 123,
  "sequence_order": 20,
  "title": "Conduct pilot review for Q4 2025",
  "owner_id": 12,
  "due_date": "2025-11-30"
}

Step 4: Track Progress

Update step status as work progresses, add comments for communication.

Step 5: Complete & Verify

PATCH /api/v1/continuous_improvement/actions/123/
{
  "status": "completed",
  "completed_at": "2025-12-01T10:00:00Z"
}

POST /api/v1/continuous_improvement/verifications/
{
  "improvement_action_id": 123,
  "outcome": "verified",
  "effectiveness_score": 5
}

2. üòü Complaint Resolution

Handle student/stakeholder complaints systematically.

Step 1: Log Complaint

POST /api/v1/continuous_improvement/actions/
{
  "title": "Trainer communication issue - Student complaint",
  "description": "Student reports lack of timely feedback from trainer.",
  "source": "complaint",
  "priority": "high"
}

Step 2: Investigation Steps

Create steps for investigation, root cause analysis, and resolution.

Step 3: Implement Resolution

Execute corrective actions and document outcomes.

Step 4: Follow-up

Verify student satisfaction and prevent recurrence.

3. üîç Process Improvement Initiative

Proactive improvement of training/assessment processes.

Step 1: Identify Opportunity

POST /api/v1/continuous_improvement/actions/
{
  "title": "Streamline assessment submission process",
  "description": "Current 5-step process can be reduced to 3 steps with online portal.",
  "source": "self_assessment",
  "priority": "medium"
}

Step 2: Design Solution

Create workflow steps for design, development, testing, and rollout.

Step 3: Pilot Implementation

Test with small cohort before full deployment.

Step 4: Full Rollout

Deploy to all students with training for staff.

Step 5: Monitor Effectiveness

Track metrics and verify improvement achieved.

4. ‚è∞ SLA Breach Response

Handle overdue actions systematically.

Step 1: Receive Alert

System sends notification when action approaches/breaches SLA.

Step 2: Assess Status

GET /api/v1/continuous_improvement/actions/456/

Review progress, identify blockers.

Step 3: Re-prioritize or Extend

PATCH /api/v1/continuous_improvement/actions/456/
{
  "priority": "critical",
  "assigned_to_id": 5  // Escalate to senior staff
}

POST /api/v1/continuous_improvement/comments/
{
  "improvement_action_id": 456,
  "body": "Escalated due to SLA breach. @senior.manager please review urgently."
}

5. üìä Compliance Monitoring

Regular review of CIR for compliance status.

Step 1: Dashboard Review

GET /api/v1/continuous_improvement/actions/compliance_overview/

Step 2: Identify Gaps

Check critical clauses without linked actions.

GET /api/v1/continuous_improvement/clause-links/by_clause/?clause_number=1.8

Step 3: Create Preventive Actions

Address gaps proactively before audit.

6. üîÑ Periodic Register Review

Quarterly/annual CIR review process.

Step 1: Generate Review Report

GET /api/v1/continuous_improvement/kpi-snapshots/?snapshot_date_after=2025-01-01

Step 2: Analyze Trends

Review completion rates, overdue patterns, category distribution.

Step 3: Identify System Improvements

Create actions for CIR process improvements.

Step 4: Document Review

Create ImprovementReview record with findings and recommendations.
